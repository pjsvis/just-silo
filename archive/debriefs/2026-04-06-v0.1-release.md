## Key Decisions

### 1. Namespace-prefixed Commands
- `dev-*` for project development
- `silo-*` for silo operations
- Clear separation, easy discovery

### 2. Mentational Hygiene
- Added directive to prevent doom loops
- 2 attempts max, then skip and report
- Nested directory signal = evidence, not task

### 3. Silo Lexicon
- JSONL format for conceptual tokens
- Emergent vocabulary capture
- `just lex`, `just lex-check`

### 4. GNU CLI Conventions
- `-h/--help` in scripts
- `--version` flag
- positional-arguments for natural CLI

## What Worked

1. **Justfile as scratchpad** — Commands added as needed
2. **Integration tests first** — Caught flush recipe bug
3. **Test-driven development** — Fixed, tested, repeated
4. **Namespace prefixes** — Clear organization

## What Didn't Work

1. **TD database corruption** — Known issue, skipped per directive
2. **Nested `.todos/.todos/` signal** — Took too long to recognize
3. **Multiple template directories** — `template/` and `templates/basic/` confusing

## Lessons Learned

### Doom Loop Prevention
```
IF same command + same error = 2nd attempt → STOP
```

### Nested Directory Signal
`.foo/.foo/` = Evidence of previous attempts. Stop investigating.

### Tidy-First
Archive empty/duplicate briefs. Keep backlog lean.
