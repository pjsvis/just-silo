# just-silo Strategy

*Positioning, pricing, and go-to-market for just-silo.*

---

## The Problem We Solve

**AI agents lose context every turn.** Every session starts from scratch. Prompt injection becomes ritual.

Existing solutions:
- **Wikis** — Static, out of date, not actionable
- **Prompt libraries** — Noise, no execution
- **Agent frameworks** — Heavy, opinionated, require commitment

**just-silo: Context lives in the filesystem. Agents drop in and work.**

---

## Positioning

### Tagline
> *"just-silo it or just-forget it"*

### Positioning Statement
> For AI practitioners who need agents to remember, just-silo is a directory-based skill framework that persists context in the filesystem. Unlike wikis or prompt libraries, just-silo makes context actionable — agents execute the rules directly.

### Target Audience
1. **AI builders** — Building multi-agent systems
2. **DevOps engineers** — Automating infrastructure with AI
3. **Data engineers** — Pipelines that AI agents can operate
4. **Researchers** — Reproducible AI workflows

---

## Competitive Landscape

| Approach | Pros | Cons |
|----------|------|------|
| Wikis | Familiar | Static, not actionable |
| Prompt libraries | Shareable | No execution |
| Agent frameworks | Powerful | Heavy, opinionated |
| just-silo | Light, filesystem-native, universal | New, requires discipline |

**Differentiation:** No runtime. No framework. Just directories with rules.

## Pricing

**Free. Open source. CC0.**

Rationale:
- We're building a standard, not a product
- Value accrues to the ecosystem
- The "proprietary workflow" moat is imaginary
- Someone will do it anyway; better it be good

**Revenue model:** Consulting, custom templates, enterprise support (future).

## Key Messages

1. **The filesystem is the interface.**
   - Context persists in files
   - No database, no server, no ceremony

2. **Bounded context is enough.**
   - Past a threshold, more context = more noise
   - Silo = just enough to act

3. **Drop in and work.**
   - Agents don't need hand-holding
   - Read rules. Do job. Done.

4. **Wee-stories expose contradictions.**
   - If you can't write it, you don't understand it
   - Docs stay coherent

## Risks

| Risk | Mitigation |
|------|------------|
| Too abstract | Concrete examples: grain silo, PR reviewer |
| Too simple | Gamma loops, sandboxing show depth |
| Too novel | Frame as "gitignore for AI context" |
| Not sticky | Lock-in via habit, not tech |
